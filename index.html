Conceptos base

Entendé cómo funciona Claude antes de arrancar

Tres conceptos que aparecen constantemente en toda la guía. Entenderlos bien desde el principio hace que todo lo demás tenga más sentido — y te evita confusión cuando leas sobre MCPs, agentes y automatización.

01conceptoChat · Cowork · Claude Code — ¿cuándo usar cada uno?
+

Claude no es solo un chatbot. Es un sistema con tres modos de interacción distintos, cada uno pensado para un tipo de trabajo diferente. La mayoría de la gente solo conoce el Chat — pero Cowork y Claude Code son donde pasa la magia real.

ModoQué hace exactamenteIdeal paraRequiere
💬 Chat
claude.ai o app		Conversación directa. Podés subir archivos, PDFs, imágenes. Los resultados aparecen en un panel lateral (Artifacts) que podés descargar o copiar.Preguntas, redacción de correos, análisis de documentos, brainstorming, resúmenesSolo navegador — plan gratuito disponible
🏢 Cowork
Claude Desktop		Claude trabaja de forma autónoma mientras hacés otra cosa. Le asignás una tarea, cerrás la ventana y cuando volvés el resultado está listo. También acepta tareas programadas — todos los lunes a las 9am, por ejemplo.Reportes automáticos, organización de archivos, monitoreo, tareas recurrentesApp Desktop + plan Pro ($20/mes)
⚡ Claude Code

No te dice cómo hacerlo — lo hace directamente en tu codebase.Desarrollo, automatización, agentes, skills, deploys, scriptsTerminal + plan Pro o créditos de API

La regla simple: ¿Necesitás una respuesta rápida? → Chat. ¿Querés que trabaje solo mientras hacés otra cosa? → Cowork. ¿Tenés un proyecto con archivos y código? → Claude Code. No son excluyentes — podés combinarlos según la tarea.
Desde la doc oficial: Claude Code funciona en Terminal, VS Code, JetBrains, Desktop app y en la web — todos comparten la misma configuración y CLAUDE.md files.
02conceptoAPIs, MCPs y A2A — los 3 protocolos que mueven la IA
+
API
Vos ↔ IA
Tu app manda un mensaje, Claude responde. La base de todo.
MCP
IA ↔ Herramientas
Claude accede directo a tus archivos, calendario, DB — sin que copies y pegues.
A2A
IA ↔ IA
Múltiples agentes coordinándose entre ellos sin intervención humana.
🔑
La analogía de las llaves: Sin MCP, Claude está encerrado — le traés toda la info. Con MCP, le das las llaves de tu oficina y va a buscar lo que necesita solo.
Para instalar un MCP: claude mcp add nombre -- npx -y @paquete/mcp. Verificar con /mcp.

Módulo 01

Fundamentos y trucos clave

Antes de instalar cualquier herramienta o agente, estos son los conceptos que más impacto tienen en el día a día. No son trucos oscuros — son las funciones centrales de Claude Code que la mayoría no usa porque nadie se las explicó bien.

01comandoultrathink + Plan Mode — el combo de arquitecto
+

Dos herramientas para darle más profundidad al razonamiento de Claude antes de que ejecute cualquier cambio. Son independientes pero funcionan muy bien juntas.

ultrathink
Palabra clave que activa el nivel máximo de razonamiento en Opus 4.6 y Sonnet 4.6. Claude dedica más tokens a pensar el problema antes de responder. La doc oficial confirma que es la forma más rápida de aumentar la profundidad de análisis para una tarea puntual sin cambiar la configuración general.
Plan Mode (Shift+Tab)
Modo solo-lectura certificado por Anthropic. Claude puede explorar archivos, buscar en tu codebase e investigar en la web — pero no puede crear, editar ni borrar ningún archivo. Es la garantía de que nada se toca hasta que vos aprobés el plan.

Activar

# Dentro de una sesión Claude Code Shift+Tab # alterna Plan Mode / Auto Mode claude --permission-mode plan # iniciar en Plan Mode
Prompt — ultrathink + Plan Modeultrathink — Analizá mi proyecto completo y creá un plan detallado para [describí tu tarea]. Investigá la estructura actual, identificá todos los archivos que necesitás tocar y dame un plan paso a paso antes de escribir cualquier código.
Regla de oro: Si la tarea toca más de 3 archivos, empezá en Plan Mode. Los 30 segundos de planear te ahorran minutos de revertir cambios incorrectos.
Tip avanzado (doc oficial): Podés configurar Plan Mode como modo default para todos tus proyectos. Agregá esto a .claude/settings.json:
{"permissions": {"defaultMode": "plan"}}
También podés abrir el plan en tu editor con Ctrl+G para editarlo antes de que Claude lo ejecute.
02conceptoSub-agentes — divide y vencerás
+
¿Qué son? Trabajadores especializados que Claude crea para atacar diferentes partes de tu tarea en paralelo. Uno para frontend, otro para backend, otro para tests — todos al mismo tiempo.
Con sub-agentes
  • Trabajan en paralelo
  • Cada uno en un área
  • Hasta 4x más rápido
  • Menos errores de contexto
Sin sub-agentes
  • Todo en secuencia
  • Mezcla frontend/backend
  • Más lento en proyectos grandes
Cómo activarCuando necesites completar esta tarea, usá sub-agentes para dividir el trabajo. Lanzá agentes separados en paralelo para cada parte independiente — uno para frontend, otro para backend, otro para tests. No hagas todo secuencialmente si podés paralelizar.
Consejo oficial de Anthropic: Podés ver y crear sub-agentes personalizados con el comando /agents desde cualquier sesión de Claude Code. Cada agente puede tener su propio system prompt, acceso a herramientas específicas y hasta su propio worktree aislado para trabajar en paralelo sin conflictos.
03comandoCLAUDE.md — el manual de tu asistente
+
📋
Qué es: Archivo Markdown en la raíz de tu proyecto. Claude lo lee automáticamente al inicio de cada sesión — reglas, contexto, estándares de código, flujos de trabajo del equipo. Según la doc oficial: apuntá a menos de 200 líneas por archivo, usá instrucciones concretas y verificables, y dividí en subtemas si crece mucho.
Auto Memory (nuevo en v2.1.59+): Además de tu CLAUDE.md, Claude puede generar sus propios apuntes automáticamente. Cuando hacés una corrección o le enseñás algo ("siempre usá pnpm, no npm"), lo guarda en ~/.claude/projects/tu-proyecto/memory/ para usarlo en sesiones futuras. Para verlo y editarlo, usá el comando /memory desde cualquier sesión.

Generar con /init

/init # Claude analiza el proyecto y genera CLAUDE.md automáticamente # Si ya existe, /init sugiere mejoras sin sobrescribir

Dónde poner CLAUDE.md según el alcance

Reglas esenciales para agregar

## Reglas de calidad - SIEMPRE verificá tu trabajo antes de darlo por terminado. - Antes de implementar, investigá el código existente. - NO implementes nada si no estás 100% seguro. ## Sistema de memoria - Antes de terminar, guardá un resumen en PROGRESS.md - Al iniciar, leé ese archivo para retomar donde dejaste. - Secciones: "Completado", "En progreso", "Pendiente".
04comandocontext-mode — reducir el contexto 98%
+
El problema: Screenshot de Playwright = 56KB. Lista de issues = 59KB. En pocas llamadas el contexto se llena y Claude pierde el hilo.
La solución: context-mode reduce 98% el contexto. Ejecuta en sandbox, indexa resultados pesados (315KB → 5.4KB con BM25) y restaura la sesión automáticamente.

Instalación

/plugin marketplace add mksglu/context-mode /plugin install context-mode@context-mode

Verificar

ctx doctor # revisa runtimes y hooks ctx stats # KB ahorrados en tiempo real

Módulo 04

Agentes y automatización

Un agente es Claude con un trabajo específico, acceso a las herramientas que necesita y autonomía para ejecutarlo solo. En este módulo vas a aprender a crear agentes desde cero, conectarlos a WhatsApp, controlar tu computadora de forma remota y programar tareas que corren en la nube de Anthropic aunque tu laptop esté apagada.

01conceptoCómo crear un agente — el flujo completo
+
👷
Un agente = Claude con un trabajo específico. Tarea definida + herramientas que necesita + autonomía para ejecutar. Como contratar un especialista.
Abrir Claude Code: claude
Entrar en Plan Mode: Shift+Tab
Describir el agente con el prompt de abajo
Claude investiga y entrega un plan sin tocar nada
Shift+Tab de nuevo para ejecutar

Prompt — agente de code reviewQuiero un agente que revise mis pull requests automáticamente. Debe: 1. Leer archivos modificados en cada PR 2. Detectar bugs y problemas de seguridad 3. Módulo 05

Productividad y skills

El mayor problema de trabajar con Claude a largo plazo es la falta de memoria — cada sesión empieza de cero. En este módulo vas a ver cómo resolverlo con Obsidian, cómo convertir cualquier fuente de información en contexto listo para Claude, y cómo instalar o crear skills que extienden sus capacidades para casos de uso específicos.

01conceptoObsidian + Claude — memoria permanente
+
🧠
La analogía: La ventana de contexto de Claude es como la RAM de tu computadora — limitada, rápida y se borra al cerrar. Obsidian es el disco duro — puede guardar miles de notas, todas conectadas entre sí, y no se borra nunca. Lo mejor es que Obsidian guarda todo como archivos .md (Markdown), que es exactamente el formato nativo de Claude. Cuando Claude abre tu vault, lo lee sin ninguna traducción — como hablar en su propio idioma.
Descargar Obsidian gratis desde obsidian.md y crear un vault
claude install-skill https://github.com/kepano/obsidian-skills
Crear CLAUDE.md en la raíz del vault
Siempre abrir desde el vault: cd ~/mi-vault && claude
Al terminar la sesiónGuardá un resumen de lo que hicimos hoy en mi vault. Creá una nota en /diario con la fecha, los cambios principales, decisiones tomadas y lo que queda pendiente. Conectá con wikilinks a las notas de proyecto relevantes.
Al iniciar la sesiónAntes de empezar, revisá mis notas sobre [proyecto] en /proyectos. Leé el STATE.md y las notas más recientes en /diario. Dame un resumen de dónde quedamos y qué sigue pendiente.
